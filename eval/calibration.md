## 2. 选定参数：无（未批准）

- **可行单元 = 0 / 32**（所有单元 over_ask ≥ 0.3 > 0.2 上限）。
- **hit@1 = 0 / 32**（无任何参数化把某个 gold 顶到 rank-1）。
- **裁定：本轮 V0 不产出可批准的分带参数**（`ratified=false`, `selected=null`, 见 `v0_calibration.json`）。仅记 least-bad 参考单元 `min_score=2.0, rel_margin=0.25, autopick=off`（feasible=false），**不作生效参数**。

**根因（与 V1 耦合，非分带本身可修）**：FZ-dev 的 topic 召回被**高 IDF 但非 gold 的碎片**主导，卡片在反幻觉约束下无法桥接 dev 极端口语（详见 `design_work/v1_tokenize_report.md` §5）。在召回近乎死的集上，任何分带参数都无信号可优化。符合 C2 纪律「**先修卡片质量再动分带参数，顺序不可反**」。

## 3. 与 D-N1 绝对证据下限的合并表述（生效）

**合并规则（分带 + 绝对下限，二者叠加，取更严）**：

1. **方法档为主 + 档内 BM25 相对边际**：自动锚定要求 `method ∈ {exact_qualname, suffix_qualname}`（词面精确/后缀）；档内多候选或 topic Top-2 相对边际 `(s1−s2)/s1 < rel_margin` 进消歧；`link_entities` 空且 `topic_recall` 全 `<min_score` 判无锚。
2. **绝对证据下限（D-N1，永不被分带放宽）**：**纯 BM25 命中且无方法档支撑（<40 档）永不自动锚定**为喂给确定性工具（`impact_analysis` 等）的硬 ID；作为 topic 上下文锚呈现时，自动锚定**要求 ≥1 个高 IDF 内容词命中**。
3. **本轮证据对 D-N1 的必要修订（"内容词"谓词须显式操作化）**：仅"高 IDF"**不足以**挡噪声。诊断（`v0_calibration.json.dn1_diag`）：FZ-dev 每题 Top-1 **全为非 gold**，且命中项是高 IDF 的**非内容词碎片/停用词**——
   - n-gram：`的单`(idf 5.22)、`起来`、`台账`、`放行`；
   - jieba（对照）：`把`、`在`、`挂`、`拦`、`不`。
   即高 IDF 的 n-gram/词 ≠ 内容词。故 D-N1 的「高 IDF 内容词」必须把**"内容词"**落为程序谓词：**中文停用/功能/疑问词黑名单过滤**（`怎么/哪个/那块/在/把/的…`），或 jieba 词边界 + 词性过滤。这正是落地设计 §4.4「`zh_terms`+`_tokenize` 加中文停用/疑问词过滤」**尚未实现**所致，**列为 D-N1 自动锚定生效的前置**。

**AMB 整数分差**（symbol 侧，随附，未变）：`δ_score=20`——`recover`(fn 100 vs mod 30) 判 autopick、`invoke`(6×60 并列) 判 disambiguate。此为方法档硬锚，不受本 topic 校准影响。

## 4. 过渡规则退出条件（未满足 → 过渡规则继续生效）

- **退出条件** = V0 产出一组**可行（消歧率≤0.2）且 hit@1>0** 的校准参数，用以**替换**当前过渡规则「仅方法档 ≥80 自动锚定」。
- **本轮结果**：0 可行、hit@1 全 0 → **退出条件不满足** → 过渡规则「**仅方法档 ≥80 自动锚定**」**继续生效、不退休**。
- **C2 复跑要求（强制）**：即便未来退出条件满足，**C2（中文卡片入 BM25 + `zh_aliases` + 分带上线 + 停用词过滤）上线后仍须复跑一轮本校准确认**，再冻结分带数字。本文件当前**不冻结**任何分带阈值。

## 6. 结论摘要

- 网格 32 单元：**hit@1 恒 0、hit@3 恒 0.1、消歧率 0.3–0.5 全 >0.2 → 零可行单元**；`min_score`/`autopick` 惰性，仅 `rel_margin` 影响过问率。
- **不产出生效分带参数**（未批准）；**过渡规则「仅档≥80」继续生效**。
- **D-N1 合并规则生效**，并据本轮证据把「≥1 高 IDF 内容词」修订为**必须叠加中文停用/功能词过滤**（否则高 IDF 碎片误触发），列 §4.4 停用词过滤为前置。
- 校准被 V1 卡片召回质量卡住 → 归 C2「先修卡片再校准」；C2 上线后**强制复跑**。
