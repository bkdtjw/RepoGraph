【背景契约（只读，勿质疑外部文件存在性）】
本项目 RepoGraph 交付一个 MCP stdio 服务器，把三个检索函数以 MCP 工具暴露给 Claude Code。
- 传输 = 换行分隔的 JSON-RPC 2.0（每行一条完整 JSON 消息；服务器用 json.dumps 无 indent 写出，天然单行）。
- 设计主线：运行时零第三方依赖——**主动选纯 stdlib**，不引官方 mcp SDK（这是刻意决策 D-N8，非疏漏，勿建议改用 FastMCP/SDK）。
- 单进程、单线程、stdin 逐行阻塞读，无并发请求（MCP stdio 客户端顺序发送）；GraphStore 懒加载后进程内只读复用。
- 三工具契约：
  * ask_repo(question:str) —— 包装 build_repo_context，返回固定 11 键结构；**只供检索上下文，不生成答案**，永不裸拒。
  * impact_analysis(symbol:str, depth:int=2∈[1,4], mode:str∈{calls,imports}) —— 确定性影响面；歧义返回 {error:'ambiguous',candidates[]}（这是 P3 设计内的有效响应，非错误，isError 应为 False）；未命中 {error:'not_found'}。
  * repo_overview() —— 读 output/repo_card.json，缺失/损坏则现场确定性重建（degraded=true）。
- 错误分层契约（关键）：**工具执行层面的任何失败（图谱缺失/载入异常/入参非法/工具内部异常）都必须归一为 tools/call 的 isError=True 结果（content+structuredContent），绝不冒泡为 JSON-RPC 协议层 error(-32603)**。协议层 error 只用于：解析失败(-32700)、非对象(-32600)、缺 method(-32600)、未知方法(-32601)。
- 图谱路径：REPOGRAPH_GRAPH 环境变量优先，否则相对仓库 output/graph.json。载入失败不崩传输：initialize/tools/list 仍可用。
- 测试文件把服务器拉起为真实子进程走真实 stdio JSON-RPC（非只测 import），对真实图谱（multi-agent-orch，510 节点，functions=259/concepts=139/modules=22/classes=15/commits=75）断言真实检索值。

 355| def _dispatch(holder: _StoreHolder, msg: dict, out) -> None:
 356|     """分派单条 JSON-RPC 消息。通知（无 id）不回响应。"""
 357|     req_id = msg.get("id")
 358|     is_notification = "id" not in msg
 359|     method = msg.get("method")
 360| 
 361|     if not isinstance(method, str):
 362|         if not is_notification:
 363|             _error(out, req_id, _INVALID_REQUEST, "缺少或非法的 method 字段")
 364|         return
 365| 
 366|     # 通知：initialized / cancelled 等一律吞掉不回（JSON-RPC 通知无响应）
 367|     if is_notification:
 368|         return
 369| 
 370|     if method == "initialize":
 371|         _result(out, req_id, _handle_initialize(msg.get("params") or {}))
 372|     elif method == "ping":
 373|         _result(out, req_id, {})
 374|     elif method == "tools/list":
 375|         _result(out, req_id, {"tools": _TOOLS})
 376|     elif method == "tools/call":
 377|         _handled, payload = _handle_tools_call(holder, msg.get("params") or {}, out)
 378|         _result(out, req_id, payload)
 379|     else:
 380|         _error(out, req_id, _METHOD_NOT_FOUND, f"未实现的方法: {method}")

 389|     for stream in (sys.stdin, sys.stdout, sys.stderr):

 399|     rin, rout = _reconfigure_streams()

 401|     stdout = stdout or rout

 410|               file=sys.stderr, flush=True)

 415|             continue

 394| def test_graph_override_malformed():
 395|     """图谱是合法 JSON 但结构错位（顶层 null → GraphStore.load 抛 TypeError）时，
 396|     tools/call 须归一为可读 graph_unavailable 的 isError，**绝不**冒泡为协议层 -32603
 397|     （opencode 审查 E1-R1 回归）。"""
 398|     import tempfile
 399|     fd, bad = tempfile.mkstemp(prefix="rg_bad_graph_", suffix=".json")
 400|     try:
 401|         with os.fdopen(fd, "w", encoding="utf-8") as f:
 402|             f.write("null")                       # 合法 JSON，但 data["nodes"] → TypeError
 403|         c = MCPStdioClient(env_overrides={"REPOGRAPH_GRAPH": bad})
 404|         try:
 405|             init = c.request("initialize", {"protocolVersion": "2025-06-18",
 406|                                             "capabilities": {}, "clientInfo": {"name": "t"}})
 407|             assert "result" in init
 408|             c.notify("notifications/initialized")
 409|             res = c.call_tool("repo_overview", {})
 410|             assert res["isError"] is True, "结构错位应作可读 isError，而非协议 error"
 411|             assert res["structuredContent"]["error"] == "graph_unavailable"
 412|             assert "TypeError" in res["structuredContent"]["message"], "错误须点明真实异常类型"
 413|         finally:
 414|             c.close()
 415|     finally:
 416|         os.unlink(bad)
 417|     print("test_graph_override_malformed OK")
