## 6. 过程治理(本次事故的制度化答案)

R1 **单一事实源**:spec 变更先于或随同代码合并;三源分叉(文档 A/文档 B/代码)再次出现视为 P0 缺陷。
R2 **agent 自修协议**:编码 agent 的驱动 prompt 模板固定写入——"任何偏离 spec 的架构级改动,必须在提交信息中引用 DECISIONS 条目编号;无编号的架构改动一律不合并"。人工 review checklist 首项即查此。
R3 **门禁强制**:每次合并前 `make gate`;Phase D 后主集并入。
R4 **tag 纪律**:每 phase 末打 tag,评测报告与简历数字必须标注来源 tag。
R5 **回写时限**:phase 结束 48 小时内完成 spec 增量回写,拖欠回写阻塞下一 phase 启动。
